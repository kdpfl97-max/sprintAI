SprintAI의 사용자 조사, 설문 분석, 제품 가설과 MVP 개선 과정을 정리한 문서 모음입니다.

## 읽는 순서

1. 인터뷰와 설문에서 관찰된 문제를 확인합니다.
2. 두 조사의 근거를 통합해 제품 가설을 판정합니다.
3. 조사 실패 원인과 후속 설문 설계를 확인합니다.
4. 검증 결과가 MVP 우선순위와 용어 개선으로 어떻게 연결됐는지 추적합니다.

## 조사 자료

### 1차 조사

- 사용자 인터뷰: 5개 세션
- 사용자 설문: 13건
