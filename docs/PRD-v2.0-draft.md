# SprintAI PRD v2.0 Draft

## 제품 정의
팀의 계획과 실행 사이에서 사라지는 정보를 구조화하는 가벼운 스프린트 도구. AI는 초안 작성 보조 역할을 한다.

## 타깃 전략
사용 주체는 팀, 구매와 반복 사용 주체는 강사/교육기관으로 본다. 부트캠프 팀은 해체될 수 있으므로 반복 사용 검증은 교육기관 단위로 본다.

## P0 우선순위
1. 할 일 캡처와 스프린트 계획 확정
2. 팀원별 가능 시간과 태스크 배정 확인
3. 실행 중 누락 정보, 블로커, 마감 위험 확인

## 측정 계획
현재값을 먼저 측정한다. PM은 계획 작성 시간을 스톱워치로 측정하고, 팀원은 자기 할 일을 확인하는 데 걸리는 시간을 측정한다.

목표: PM 3분 이내, 팀원 10초 이내.

## 연동 판단
Notion/Sheets 가져오기는 P0 승격 후보. 1차 MVP는 신규 팀/교육 현장 검증을 우선하고, 반복 사용 신호가 나오면 가져오기를 P0로 올린다.

## 다음 단계
1. v2 화면 기준 대시보드 개선
2. 마감 위험 태스크 확인
3. PM 관리 액션 정리
4. 팀원 화면의 오늘 할 일 밀도 개선
