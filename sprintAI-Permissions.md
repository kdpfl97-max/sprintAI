# SprintAI 역할 & 권한 정의

## 역할 체계

| 역할 | 설명 | 권한 등급 |
|------|------|-----------|
| PM | 프로젝트 매니저. 스프린트 결정권 보유 | 관리자 |
| 프론트엔드 / 백엔드 / AI·백엔드 / 디자인 / 풀스택 / DevOps / QA | 일반 개발 팀원 | 일반 |

> 로그인 방식: 앱 시작 시 사이드바 하단에서 팀원 선택 (별도 계정/비밀번호 없음, MVP 방식)

---

## AI 스프린트 빌더 권한 상세

| 액션 | PM | 일반 팀원 | 비로그인 |
|------|:---:|:--------:|:--------:|
| 페이지 조회 (읽기) | ✅ | ✅ | ✅ |
| 태스크 선택/해제 | ✅ | ✅ | ❌ |
| Capacity 슬라이더 조정 | ✅ | ✅ | ❌ |
| **AI 분석 실행** | ✅ | 🔒 | 🔒 |
| **태스크 담당자 배정** | ✅ | 🔒 | 🔒 |
| **스프린트 확정** | ✅ | 🔒 | 🔒 |
| **스프린트 삭제/초기화** | ✅ | 🔒 | 🔒 |

> 🔒 = 버튼 비활성화 + 잠금 아이콘 표시

---

## 다른 페이지 권한 (현재 제한 없음 — 추후 확장 가능)

| 페이지 | 현재 | 추후 제안 |
|--------|------|-----------|
| 아이디어 캡처 | 모두 사용 가능 | 개인 소유 (로그인 기준 분리) |
| 백로그 | 팀 공유, 모두 편집 가능 | 삭제는 PM만 |
| 칸반 보드 | 모두 이동 가능 | 자기 담당 태스크만 이동 |
| 팀 관리 | 모두 접근 가능 | 추가/삭제는 PM만 |
| 대시보드 | 읽기 전용 (모두) | 유지 |

## 구현 파일

- `src/store/useAuthStore.js` — 현재 로그인 유저, `can` 권한 객체
- `src/components/layout/Sidebar.jsx` — 팀원 선택 드롭다운 (하단 유저 영역)
- `src/pages/SprintBuilderPage.jsx` — 권한 제한 적용

## can 객체 구조

```js
can = {
  runAI:         boolean,  // AI 분석 실행
  confirmSprint: boolean,  // 스프린트 확정
  assignMember:  boolean,  // 담당자 배정
  deleteSprint:  boolean,  // 초기화
}
// 현재 PM 역할만 true, 나머지 false
```
