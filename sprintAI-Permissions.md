## 대시보드 — 역할별 분리 화면

### PM 홈 (isPM === true)

PM 대시보드는 팀 전체 상태 파악과 위험 요소 확인에 집중한다.

| 섹션 | 내용 |
|------|------|
| 핵심 지표 3개 | 완료율, 남은 기간, 확인 필요 건수 |
| 확인이 필요한 항목 | 미배정 / 블로커 / 마감 임박 / 대기 중 태스크 多 |
| 팀원별 남은 작업 시간 | 바 차트 + 가용 시간 대비 과부하 배지 |
| 번다운 차트 | 칸반 보드 링크 포함 |

### 팀원 홈 (isPM === false)

팀원 대시보드는 "내 다음 행동"에 집중한다.

| 섹션 | 표시 조건 | 내용 |
|------|-----------|------|
| 새 스프린트 배정 알림 배너 | startedAt 기준 48시간 이내 | 새로 배정된 내 업무 목록 |
| 내 진행 요약 | 항상 | 아바타 + 내 배정 업무 수 + 완료율 |
| 지금 할 일 / 다음 할 일 | 항상 | 2열 그리드 |
| 의존 관계 | 의존 업무 있을 때만 | 나를 기다리는 / 내가 기다리는 업무 |
| 이번 주 마감 | 마감 임박 업무 있을 때만 | dueDate 기준 3일 이내 |
| 팀 전체 진행률 | 항상 | 전체 완료율 바 |

## 페이지별 권한

| 페이지 | 현재 | 추후 제안 |
|--------|------|-----------|
| 전체 할 일 (구 백로그) | 팀 공유, 모두 편집 가능 | 삭제는 PM만 |
| 이번 계획 만들기 (구 AI 스프린트 빌더) | PM만 AI 초안·확정·배정 가능 | 유지 |
| 칸반 보드 | 모두 상태 이동 가능 | 자기 담당 태스크만 이동 |
| 대시보드 | 역할별 분리 화면 표시 | 유지 |
| 팀 관리 | 모두 접근 가능 | 추가/삭제는 PM만 |
| 회고 | 팀원 4L 입력, PM만 AI 회고 초안 만들기 | 유지 |

## 구현 파일

- `src/store/useAuthStore.js` — 현재 로그인 유저, `can` 권한 객체
- `src/store/useBacklogStore.js` — 전체 할 일 상태 관리 (STORAGE_KEY: `sprintai_backlog_v2`)
- `src/store/useSprintStore.js` — 스프린트 태스크 상태 관리
- `src/components/layout/Sidebar.jsx` — 팀원 선택 드롭다운 (하단 유저 영역)
- `src/pages/SprintBuilderPage.jsx` — 이번 계획 만들기, 권한 제한 적용
- `src/pages/DashboardPage.jsx` — PM/팀원 역할별 분리 렌더링 (`isPM` 플래그 기준)
- `src/pages/RetroPage.jsx` — 팀원 4L 입력 강조, PM만 AI 회고 초안 만들기 버튼 표시

## can 객체 구조

```js
can = {
  runAI:         boolean,  // AI 계획 초안 만들기 실행
  confirmSprint: boolean,  // 스프린트 확정
  assignMember:  boolean,  // 담당자 배정
  deleteSprint:  boolean,  // 초기화
}
// 현재 PM 역할만 true, 나머지 false
```

## 프라이버시 원칙

사용자 조사(2026-06-28)에서 확인된 원칙으로, 구현 전반에 적용한다.

- 개인의 접속 시간·체류 시간을 수집하거나 표시하지 않는다.
- 팀원 순위와 개인 생산성 점수를 만들지 않는다.
- 과부하 알림은 개인 평가가 아니라 업무 재배분을 위한 신호로만 사용한다.
- 팀원도 자신에 관한 예상 시간 산정 기준을 확인할 수 있어야 한다.
