## 핵심 결론

SprintAI의 다음 개발 단계는 다음 두 기준을 실제로 증명하는 데 집중한다.

> **PM은 회의 전에 팀 상태를 3분 안에 파악하고, 팀원은 묻지 않고도 자신의 다음 할 일을 찾을 수 있다.**

### 조사에서 확인된 핵심 문제

1. 팀장은 팀원별 진행 상황과 가용 시간을 일일이 물어봐야 한다.
2. 팀원은 맡은 일을 끝낸 후 다음에 무엇을 할지 알기 어렵다.
3. 구두 공유만으로는 업무 누락과 상태 불일치가 발생한다.
4. Jira·Notion은 강력하지만 초기 설정과 운영 비용이 부담스럽다.
5. AI 제안은 근거를 이해하고 수정할 수 있을 때만 신뢰받는다.

### 이번 MVP 수정에서 반영한 내용

| 인사이트 | 반영된 기능 |
|---|---|
| 용어 직관화 | 백로그→전체 할 일, 스프린트 빌더→이번 계획 만들기, AI 분석→AI 계획 초안 만들기 |
| 작업량 분리 | `작업량` → `예상 시간(시간)` + `난이도(낮음/보통/높음)` |
| 팀원 홈 재구성 | 지금 할 일 / 다음 할 일 / 의존 관계 / 이번 주 마감 순서로 구성 |
| PM 대시보드 | 일반 통계보다 "확인이 필요한 항목" 먼저 표시 |
| AI 신뢰 보완 | 추천 이유 표시, 확인 필요 항목 배지, 이동 미리보기 |
| 블로커 가시화 | 칸반 카드 빨간 테두리 + 선행 업무 이름/담당자 표시 |
| 산출물 연결 | 태스크별 outputLink 필드 + 칸반 카드에서 등록 가능 |
| 프라이버시 원칙 | 접속 시간·생산성 점수 없음, 과부하 알림은 재배분 신호로만 사용 |
